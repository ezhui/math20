# 小兔子跳格子 — 20以内加减法 HTML 游戏 设计文档

## Context

为 4 岁半孩子设计一个单文件 HTML 数学游戏，用「正数/倒数/跳格子」可视化 20 以内加减法。孩子在数字条上看到小兔子一步步跳跃，直观理解加法就是往后数、减法就是往前数。全程点击操作，无需打字。

---

## 1. 产品概述

| 项目 | 说明 |
|------|------|
| 目标用户 | 4 岁半幼儿 |
| 能力范围 | 20 以内加减法 |
| 核心隐喻 | 加法 = 往后跳（正数），减法 = 往前跳（倒数） |
| 交互方式 | 纯点击，无需键盘 |
| 技术形态 | 单文件 HTML，双击即用 |

---

## 2. 游戏流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  开始页   │───▶│ 选运算类型 │───▶│  选难度   │───▶│  自动演示  │───▶│  选择答案  │───▶│  结果反馈  │
│ (大标题)  │    │ 加法/减法 │    │ 简单/挑战 │    │ (兔子跳格) │    │ (点按钮)  │    │ (鼓励语)  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
                                                                                     │
                                                     ┌───────────────────────────────┘
                                                     ▼
                                              ┌──────────┐
                                              │  下一题   │
                                              └──────────┘
```

### 详细步骤

1. **开始页**：大标题"小兔子跳格子"，两只兔子招手动画，两个超大栏目卡片左右排布：

   | ➕ 加法 | ➖ 减法 |
   |---------|---------|
   | 暖橙色卡片 | 冷蓝色卡片 |
   | 🐰 往后跳 | 🐰 往前跳 |
   | 数字越来越大 | 数字越来越小 |
   | 副标题"正数！" | 副标题"倒数！" |

   点击卡片 → 进入该运算模式

2. **选难度**：进入运算模式后，显示两个难度按钮
   - 🐣 **简单模式**：数字范围 1~10
   - 🐰 **挑战模式**：数字范围 1~20
   - 无论哪种模式，数字条始终显示完整的 1~20
3. **自动出题**：全程只出该运算类型的题目（如选了加法，永远不出减法题），根据所选难度控制数字范围
3. **自动演示**：小兔子从起始数字出发，一格一格跳，每跳一格停顿 0.5-0.8 秒，同时高亮当前格、念出数字（视觉化）
4. **选择答案**：演示结束后，显示 4 个选项按钮（含正确答案 + 3 个干扰项），孩子点击选择
5. **结果反馈**：
   - 答对：撒花/星星动画 + 语音鼓励（"太棒了！🎉"）
   - 答错：温和提示"差一点点，再试一次～"，不批评，可重选
6. **下一题**：自动过渡到下一题

## 4. 交互细节

### 4.1 按钮规范

- 最小触摸区域：`80px × 80px`（适合幼儿手指）
- 圆角：`20px`
- 按下有缩放反馈（`transform: scale(0.9)`）
- 悬停有阴影增强

### 4.2 答错处理

- 不显示红色/叉号（避免负面情绪）
- 兔子歪头说"差一点点，再试一次～"
- 选错的按钮变灰，减少干扰
- 不扣分，不记录错误

### 4.3 答对处理

- 全屏撒花/星星粒子动画
- 兔子跳起欢呼
- 得分 +1 星
- 1.5 秒后自动出下一题

## 5. 技术实现

### 5.1 文件结构

单文件 `jump-math.html`，包含：
- 内嵌 `<style>` — 所有 CSS（约 300 行）
- 内嵌 `<script>` — 所有 JS 逻辑（约 400 行）
- HTML 结构（约 100 行）

### 5.2 核心模块

```
GameState {
  score: number              // 累计星星数
  mode: 'add' | 'sub'       // 加法模式 或 减法模式
  difficulty: 'easy' | 'hard' // 简单(1~10) 或 挑战(1~20)
  currentQuestion: { a, b, op, answer }
  phase: 'start' | 'mode_select' | 'difficulty_select' | 'demo' | 'answer' | 'feedback' | 'transition'
  selectedAnswer: number | null
}
```

```
核心函数:
- generateQuestion(mode, max) // 按运算类型生成题目（不再随机混合）
- generateOptions()            // 生成 4 个选项（1正确+3干扰）
- animateJump()                // 逐步跳跃动画（加法向右、减法向左）
- checkAnswer()                // 判断对错
- showCelebration()            // 撒花动画
- resetForNext()               // 过渡到下一题
```

### 5.3 跳跃动画实现

使用 CSS `transition` + `setTimeout` 链式调用：

```javascript
async function animateJump(start, steps, direction) {
  for (let i = 0; i <= steps; i++) {
    const current = start + (direction * i);
    highlightCell(current);           // 高亮当前格
    moveBunnyTo(current);            // 移动兔子（CSS transition 0.4s）
    playPopSound();                   // 可选：轻弹音效
    await sleep(600);                 // 停顿让孩子看清
  }
}
```

### 5.4 出题算法

```javascript
function generateQuestion(mode, max) {
  // mode: 'add' | 'sub'
  // max:  10 (简单) | 20 (挑战)
  let a, b, answer;

  if (mode === 'add') {
    // 加法：a + b <= max, a >= 1, b >= 1
    a = randomInt(1, max - 1);
    b = randomInt(1, max - a);
    answer = a + b;
  } else {
    // 减法：a - b >= 1, a <= max
    a = randomInt(2, max);
    b = randomInt(1, a - 1);
    answer = a - b;
  }

  return { a, b, op: mode === 'add' ? '+' : '-', answer };
}
```

### 5.5 干扰项生成

```javascript
function generateOptions(correctAnswer) {
  const options = new Set([correctAnswer]);
  while (options.size < 4) {
    // 在正确答案 ±5 范围内生成，限制在 1~20
    const offset = randomInt(-5, 5);
    const candidate = Math.max(1, Math.min(20, correctAnswer + offset));
    options.add(candidate);
  }
  return shuffle([...options]);
}
```

### 5.6 撒花动画

使用 CSS `@keyframes` + 动态生成粒子元素：
- 随机颜色（金、粉、绿、蓝）
- 从兔子位置爆发
- 0.8 秒后消散
- 使用 `requestAnimationFrame` 或纯 CSS animation

## 7. 已确认的设计决策

1. ✅ **内置音效**：使用 Web Audio API 生成轻弹声（跳一格"啵"一声）+ 答对欢快铃声
2. ✅ **两级难度**：开始页两个大按钮 — 🐣 简单（1~10）和 🐰 挑战（1~20）
3. ✅ **完整数字条**：始终显示 1~20 全部格子，像尺子一样，帮助建立完整数感
